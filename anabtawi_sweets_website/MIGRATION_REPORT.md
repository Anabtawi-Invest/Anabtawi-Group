# Migration and Change Report

## Comparison with Module A

Module A demonstrated a valid basic addon shape: a Website dependency, manifest asset declarations, SCSS/JavaScript assets, QWeb templates, and explicit snippet registration. Module B originally omitted snippet registration and referenced frontend resources that did not exist.

Module A was used only as a structural reference. Its branding and page content were not copied. Its root controller override and core menu mutations were deliberately not copied because they are unsafe for a production Odoo 19 website.

### Input file-by-file comparison

| Relative file | Module A | Original Module B | Audit result |
|---|---:|---:|---|
| `preview.html` | 50,968 bytes | 70,121 bytes | Different standalone previews; neither was loaded by Odoo. Removed from the production addon. |
| `__init__.py` | Imported controllers | Imported controllers | B's import was removed with its unnecessary controller. |
| `__manifest__.py` | 969 bytes | 946 bytes | Both used generic `1.1` metadata; B lacked snippets and Odoo 19 theme declarations. Replaced. |
| `controllers/__init__.py` | Imported `main` | Imported `main` | Removed from B. |
| `controllers/main.py` | Two routes | Five routes | Both shadowed website routes; B also shadowed `/contactus`. Removed from B. |
| `views/website_templates.xml` | 29,607 bytes | 61,739 bytes | B combined layout, menus, and five pages, modified core menus, and contained mojibake. Split and rebuilt as Odoo page/layout records. |
| `views/snippets.xml` | Present | Missing | B received Odoo 19 snippet templates and registration. |
| `static/src/scss/website_style.scss` | 11,774 bytes | 19,255 bytes | B contained duplicated blocks and obsolete/missing-feature styles. Replaced with compiled Odoo 19 SCSS. |
| `static/src/js/website_animation.js` | 4,403 bytes | 6,943 bytes | B used the legacy widget API and requested 120 absent frames. Removed; no custom JS is required. |
| `static/src/img/*` | Nine Invest-branded images | Missing | A's branding was not copied. Twenty-five Anabtawi source images were packaged instead. |
| `static/src/video/brand_intro.mp4` | Present | Missing | A's branded video was not copied. B uses the Anabtawi public brand film embed. |
| `__pycache__/*.pyc` | Present | Present | All generated bytecode was excluded from the fixed package. |

No original file with the same relative path was byte-identical between A and B. Module B had no unique source file beyond the relative paths shared with Module A; its key problem was missing supporting structure and assets, not an extra subsystem.

## Manifest

- Changed the version from `1.1` to `19.0.1.0.0`.
- Changed the category from `Website` to the Odoo 19 theme category `Website/Theme`.
- Kept the minimal `website` dependency; no unnecessary sales, mail, or editor dependency was added.
- Kept `application` set to `True` so the custom theme is discoverable and installable from the Odoo/Odoo.sh Apps dashboard. Its `Website/Theme` category still identifies its functional type.
- Preserved `LGPL-3` and changed the author metadata to the website owner.
- Replaced the single oversized XML data entry with ordered layout, snippet, homepage, page, contact, and menu files.
- Added `web._assets_primary_variables` for Odoo-native colors and fonts.
- Removed the obsolete JavaScript asset; the remaining frontend bundle contains only valid SCSS.

## Python and controllers

- Removed `controllers/main.py` and `controllers/__init__.py`.
- Removed imports from the root `__init__.py`.
- Removed every route override, including `/`, `/aboutus`, `/branches`, `/our-catalog`, and `/contactus`.
- Removed generated `__pycache__` and `.pyc` files from the distributable.

Why: Odoo already serves `website.page` records and its standard homepage/contact pages. The old routes shadowed core routes, prevented normal page management, and risked conflicts with other modules.

## Website pages and QWeb

- Implemented About, Branches, and Catalog as published `website.page` records.
- Implemented the homepage by inheriting the existing `website.homepage` page view instead of overriding `/`.
- Implemented the contact page by inheriting `website.contactus` and retaining its standard form context.
- Added `data-name` and `data-snippet` metadata to editable page sections.
- Replaced the original corrupted Arabic mojibake with valid UTF-8 English content matching the default language of the reference site.
- Split the former 748-line template file into focused files.
- Kept QWeb expressions limited to Odoo-supported `t-call`, `t-foreach`, `t-as`, `t-att-*`, and `t-out` constructs.

## Menus

- Removed writes to `website.menu_home` and `website.menu_contactus`.
- Preserved Odoo's standard Home and Contact menus.
- Added only three module-owned menu records: About, Branches, and Catalog.
- Linked each custom menu to its related `website.page`.

Why: modifying or detaching core menus creates uninstall, multi-module, and future-upgrade conflicts.

## Layout

- Added the reference site's contact strip and floating social links through a `website.layout` inheritance.
- Replaced the standard footer at Odoo 19's documented `//div[@id='footer']` target.
- Added secure `rel="noopener noreferrer"` values to external links that open a new tab.
- Corrected the telephone URI so the displayed and dialed numbers agree.
- Removed the non-functional newsletter field.

## Contact form

- Replaced the inert `action="#"` form with Odoo's `/website/form/` endpoint.
- Used the standard `mail.mail` website-form action, required field classes, model field names, prefill behavior, submit class, and thank-you redirect.

## Snippets

- Added an Anabtawi Sweets snippet group using Odoo 19's `website.snippets` structure.
- Added editor-ready Hero and Product Categories snippets with thumbnails, names, groups, and search keywords.
- Used the Odoo 19 `snippet_groups` and `snippet_structure` XPath targets confirmed in official source.

## Assets and imagery

- Removed all image hotlinks to `anabtawisweets.com` and Unsplash.
- Packaged the logo, decoration, eight category images, three news images, three about images, two branch maps, six certificates, and payment image locally.
- Removed the 120-frame canvas implementation because none of its frame files existed.
- Retained the reference brand film as a privacy-enhanced YouTube embed with lazy loading.

## SCSS

- Removed the remote SCSS `@import`, duplicated animation blocks, nonexistent canvas/frame styles, and legacy Bootstrap-era selectors.
- Added an Odoo 19 color palette and Cairo font configuration in the primary variables bundle.
- Rebuilt frontend styles with Bootstrap 5-compatible utilities and scoped Anabtawi classes.
- Added responsive behavior, logical properties for RTL compatibility, visible keyboard focus behavior, and reduced-motion handling.

## JavaScript

- Removed the deprecated `@web/legacy/js/public/public_widget` dependency.
- Removed console logging and listeners that were never needed for the final site.
- Removed the broken preloader that counted failed images as successful loads and could display a blank canvas.
- No replacement JavaScript is required; Odoo handles menus, forms, editing, video embedding, and responsive behavior natively.
